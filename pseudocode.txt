Balance-Preserving Normalization Algorithm (Proportional Normalization)
=======================================================================

PURPOSE:
  Enforce the transportation balance constraint after Karnik-Mendel
  defuzzification: sum(supply) == sum(demand)

ROOT CAUSE:
  The fuzzifier uses different reliability formulas for supply vs demand,
  causing their KM centroids to shift by different percentages. This breaks
  the original balance even when the problem was perfectly balanced.

ALGORITHM:
  Input:  supply_km  — KM-defuzzified supply vector (length m)
          demand_km  — KM-defuzzified demand vector (length n)

  Step 1: Compute current totals
    T_supply ← sum(supply_km)
    T_demand ← sum(demand_km)

  Step 2: Compute symmetric anchor (equidistant between the two totals)
    T_anchor ← (T_supply + T_demand) / 2

  Step 3: Scale each vector proportionally to sum to T_anchor
    scale_s ← T_anchor / T_supply
    scale_d ← T_anchor / T_demand

    FOR each i in 0..m-1:
        supply_final[i] ← supply_km[i] * scale_s

    FOR each j in 0..n-1:
        demand_final[j] ← demand_km[j] * scale_d

  Step 4: Verify exact balance
    assert |sum(supply_final) - sum(demand_final)| < tolerance
    assert |sum(supply_final) - T_anchor| < tolerance

  Output: supply_final, demand_final, balance_report

MATHEMATICAL GUARANTEES:
  1. Exact balance:   sum(supply_final) == sum(demand_final) == T_anchor
  2. Ratio preserved: supply_final[i] / supply_final[k]
                     == supply_km[i] / supply_km[k]   for all i, k
  3. Deterministic:   no randomness, fully reproducible
  4. Minimal change:  correction magnitude is O(|T_supply - T_demand|)
                      which is < 0.02% for standard parameters
